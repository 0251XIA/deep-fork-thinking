# 中复杂级 · 深度分叉法

**适用条件（满足 ≥3 项）**：
- 分叉点 2~3 个
- 相关方 3~5 人
- 周期 1~4 周
- 潜在利益冲突
- 部分可逆

---

**执行流程**：
```
Step 3（澄清）→ 复杂度评估 → 加载中复杂级
        ↓
Step 3a 追问（2~3层）→ Step 3b 挖本质 → Step 4 剃刀精简
```

---

## Step 3a：苏格拉底追问（2~3层）

**追问框架**（逐轮追问 + 随时检查终止）：
- 第1轮：为什么 [分叉点] 会影响结果？
- 第2轮：这个前提成立吗？如果不成立，本质是什么？
- 第3轮：还有什么更深的原因？
（每轮结束后检查终止条件是否满足）

**追问终止**：用户回答中出现"本质是...""归根结底是...""核心是..." → 停止

**示例**（统一分叉点：是否接受新任务）：
```
Q: 为什么接受新任务会影响工作安排？
A: 会耽误现有的本职工作

Q: 耽误本职一定导致我拒绝吗？这个前提成立吗？
A: 不一定，如果任务价值足够高

Q: 去掉"耽误本职"这个前提，本质是什么？
A: 本质是剩余产能与任务价值的匹配
→ 终止（出现"本质是"）
```

---

## Step 3b：第一性原理

**追问**：
- 抛开现有流程/惯例，最基础的原理是什么？
- 如果从零开始设计，我会怎么做？

**示例**（统一分叉点：客户投诉处理）：
```
Q: 抛开现有的投诉处理流程，最基本的逻辑是什么？
A: 责任定性 → 影响评估 → 补偿决策

Q: 如果从零开始设计，你会怎么做？
A: 先判断责任归属，再根据客户影响力确定补偿力度
```

---

## Step 4：整合输出（标准）

**决策树**：

```
最终结果：[目标]
│
├─ 分叉点1：[因素名]
│  ├─ 取值A → 本质：[本质] → 行动：[1. 2.]
│  └─ 取值B → 本质：[本质] → 行动：[1. 2.]
│
└─ 分叉点2：[因素名]
   └─ ...
```

---

## Step 5 后：总结概括

> ⚠️ **无论复杂度等级，输出决策树后必须进行总结概括**

**总结话术**：
> 让我来总结一下这次分析的核心内容：
>
> **背景**：……（问题的来龙去脉）
>
> **核心发现**：……（通过追问挖到的本质，不是表象）
>
> **关键洞察**：……（2~3个最重要的判断）
>
> **行动方向**：……（用户应该怎么做）

**总结格式**：
```
【总结】
背景：……
核心发现：……
关键洞察：……
行动方向：……
```

---

## 一句话总结

**中复杂级核心**：追问触及本质，深挖价值匹配，输出结构清晰的决策树。
