# 普通级 · 深度分叉法

**适用条件（满足 ≥3 项）**：
- 分叉点 1~2 个
- 相关方 ≤2 人
- 周期 ≤7 天
- 无利益冲突
- 可逆

---

**执行流程**：
```
Step 3（澄清）→ 复杂度评估 → 加载普通级
        ↓
Step 3a 追问（1~2层）→ Step 3b 挖本质 → Step 4 剃刀精简
```

---

## Step 3a：苏格拉底追问（1~2层）

**追问框架**（逐轮追问 + 随时检查终止）：
- 第1轮：为什么 [分叉点] 会影响结果？
- 第2轮：这个前提一定成立吗？
（每轮结束后检查终止条件是否满足）

**追问终止**：用户回答中出现"不一定""如果...就不同了""未必" → 停止

**示例**（统一分叉点：是否接受新任务）：
```
Q: 为什么接受新任务会影响工作安排？
A: 会耽误现有的本职工作

Q: 这个前提一定成立吗？
A: 不一定，如果本职工作本来就不饱和
→ 终止（出现"不一定"）
```

---

## Step 3b：第一性原理（简化）

**简化版追问**：抛开现有做法，这件事最基本的逻辑是什么？

**示例**（统一分叉点：是否接受新任务）：
```
Q: 抛开现有的判断方式，这件事最基本的逻辑是什么？
A: 投入（时间精力）vs 回报（价值/关系），值不值的问题
```

---

## Step 4：整合输出（简化）

**决策树**：

```
最终结果：[目标]
│
├─ 分叉点1：[因素名]
│  ├─ 取值A → 行动：[最小行动]
│  └─ 取值B → 行动：[最小行动]
└─ ...
```

---

## Step 4 后：总结概括

> ⚠️ **无论复杂度等级，输出决策树后必须进行总结概括**

**总结话术**：
> 让我来总结一下这次分析的核心内容：
>
> **背景**：……（问题的来龙去脉）
>
> **核心发现**：……（通过追问挖到的本质，不是表象）
>
> **关键洞察**：……（1~2个最重要的判断）
>
> **行动方向**：……（用户应该怎么做）

**总结格式**：
```
【总结】
背景：……
核心发现：……
关键洞察：……
行动方向：……
```

---

## 一句话总结

**普通级核心**：快速识别分叉点，找到反例即停追问，直接给出最小行动。
