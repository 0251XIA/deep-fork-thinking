# 高复杂级 · 深度分叉法

**适用条件（满足 ≥3 项）**：
- 分叉点 ≥4 个
- 相关方 ≥6 人
- 周期 ≥1 月
- 明显利益冲突
- 不可逆

---

**执行流程**：
```
Step 3（澄清）→ 复杂度评估 → 加载高复杂级
        ↓
Step 3a 追问（3~5层）→ Step 3b 挖本质 → Step 4 剃刀精简 + ECPBMR
```

---

## Step 3a：苏格拉底追问（3~5层）

**追问框架**（逐轮追问 + 随时检查终止）：
- 第1轮：为什么 [分叉点] 会影响结果？
- 第2轮：这个前提成立吗？如果不成立，本质是什么？
- 第3轮：还有什么更深的原因？换一个人做会不同吗？
- 第4轮+：这反映了什么根本矛盾？
（每轮结束后检查终止条件是否满足）

**追问终止**：用户无法再追问更底层的原因（开始用"规则/流程/制度"来解释，而非继续回答"为什么"）；或明确指出这是组织/制度层面的限制（"这是规定""这是政策/法律限制""这是权力结构决定"） → 停止

**示例**（统一分叉点：跨部门协作难）：
```
Q: 为什么跨部门协作难会影响项目进度？
A: 各部门有各自优先级，不愿意配合

Q: 各部门不愿意配合，这个前提一定成立吗？
A: 不一定，如果配合能给部门带来直接好处

Q: 去掉"不愿意配合"这个前提，本质是什么？
A: 本质是利益诉求不统一，配合成本大于收益

Q: 这个利益不统一由什么决定的？换一个人协调会不同吗？
A: 取决于组织结构和考核机制，不是个人能力问题

Q: 这反映了什么根本矛盾？
A: 这是个人理性与组织理性之间的结构性冲突——部门最优不等于组织最优
```

---

## Step 3b：第一性原理

**追问**：
- 抛开现有流程/惯例，最基础的原理是什么？
- 如果从零开始设计，我会怎么做？
- 现有做法有多少是"一直这么做"而非"必须这么做"？

**示例**（统一分叉点：跨部门协作难）：
```
Q: 抛开现有的协作流程，这件事最基本的逻辑是什么？
A: 谁受益谁推动，阻力来自收益不均

Q: 如果从零开始设计，你会怎么做？
A: 建立利益绑定机制，让配合变成各部门的最优解

Q: 现有做法有多少是"一直这么做"而非"必须这么做"？
A: 70%的跨部门协调会是惯例而非必要的
```

---

## Step 4：整合输出（完整 + ECPBMR）

**决策树**：

```
最终结果：[目标]
│
├─ 分叉点1：[因素名]
│  ├─ 取值A → 本质：[深层本质] → 行动：[分阶段行动]
│  └─ 取值B → 本质：[深层本质] → 行动：[分阶段行动]
│
├─ 分叉点2：[因素名]（嵌套）
│  └─ ...
│
└─ ECPBMR 执行框架
```

**ECPBMR 框架**：

| 字母 | 含义 | 内容 |
|------|------|------|
| E | End（终局） | 量化目标：______ |
| P | Path（路径） | 分叉树：______ |
| B | Breakthrough（突破） | 关键节点：______ |
| M | Move（执行） | 谁+何时+做什么：______ |
| R | Review（复盘） | 7天/30天/60天检查点：______ |

**⭐ECPBMR 嵌入规则**：
- 在决策树中找到关键节点：该分支的取值决定了后续其他分支是否还有意义
- 例如"高层支持弱"分支：如果不先解决高层背书，后续部门配合和资源到位都无从谈起
- 该节点标注 `⭐ECPBMR`，在决策树下方输出 ECPBMR 框架
- 不是所有分支都标注，只标需要集中资源攻克的"路径断裂点"

**决策树输出示例**：
```
最终结果：项目按时上线
│
├─ 高层支持
│  ├─ 强 → 正式文件+指标分解+定期汇报
│  └─ 弱 → 找利益结合点+让部门自主要参与  ⭐ECPBMR
│
└─ Deadline
   ├─ 可调 → 分阶段+甜点先行
   └─ 硬 → 集中资源攻关键

⭐ECPBMR（针对"高层支持弱"分支）：
E: 项目Q2前上线
P: 高层支持→部门配合→资源到位
B: 高层背书是关键突破点
M: 本周完成高层汇报，下周启动部门对齐
R: 7天检查点/30天里程碑/60天复盘
```

---

## Step 4 后：总结概括

> ⚠️ **无论复杂度等级，输出决策树后必须进行总结概括**

**总结话术**：
> 让我来总结一下这次分析的核心内容：
>
> **背景**：……（问题的来龙去脉）
>
> **核心发现**：……（通过追问挖到的本质，不是表象）
>
> **关键洞察**：……（2~3个最重要的判断）
>
> **行动方向**：……（用户应该怎么做）

**总结格式**：
```
【总结】
背景：……
核心发现：……
关键洞察：……
行动方向：……
```

---

## 一句话总结

**高复杂级核心**：深度追问挖到结构性矛盾，第一性原理剥离假设，完整决策树 + ECPBMR 执行框架确保落地。
